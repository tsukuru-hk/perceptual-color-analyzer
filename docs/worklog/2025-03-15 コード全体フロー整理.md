# 2025-03-15 コード全体フロー整理

コードの流れを追って、画像アップロードから OKLCH 表示までのデータフローとアーキテクチャを図解した。

---

## 全体のデータフロー

```
ユーザーが画像を選択
  → Canvas に描画
    → クリックした座標の RGB を取得
      → Culori で OKLCH に変換
        → L(明度)・C(彩度)・h(色相) を表示
```

---

## コンポーネント構成

```
App.vue
└── ImageAnalysisContainer.vue  ← 状態管理とユースケース呼び出し
     ├── <input type="file">    ← 画像選択
     ├── ImageCanvas.vue        ← Canvas 描画 & クリック座標取得
     └── PixelOklchResult.vue   ← OKLCH 値の表示
```

---

## フェーズ1: 画像の読み込み

```
ユーザーがファイルを選択
        │
        ▼
ImageAnalysisContainer.vue
  onFileChange(e)
  loadProgress = 'loading'
        │
        ▼
application/useCase/loadImageUseCase.ts
  loadImageToImageData(file, maxW, maxH) を呼ぶ
        │
        ▼
infrastructure/imageLoader.ts
  ① loadFileAsImage(file)
     URL.createObjectURL(file) → Image 要素に読み込む
  ② drawToCanvasAndGetImageData(img, maxW, maxH)
     オフスクリーン Canvas 作成
     → アスペクト比を維持してリサイズ描画
     → ctx.getImageData() でピクセルデータ取得
     → Result<ImageData, ImageLoadError> を返す
        │
        ▼
ImageAnalysisContainer.vue
  成功 → imageDataRef = result.value / loadProgress = 'done'
  失敗 → errorMessage を表示
```

---

## フェーズ2: Canvas 描画

```
imageDataRef が更新される
        │
        ▼
ImageCanvas.vue
  watch(imageData) → nextTick() 後に drawCanvas()
  canvas.width / height = data のサイズ
  ctx.putImageData(data, 0, 0)  ← 画面に画像を描画
```

---

## フェーズ3: ピクセルクリック → OKLCH 変換

```
ユーザーが Canvas をクリック
        │
        ▼
ImageCanvas.vue
  handleClick(e):
    CSS 表示サイズ → 実ピクセル座標に変換
    scaleX = canvas.width / rect.width
    x = (e.clientX - rect.left) * scaleX
    → props.onCanvasClick(x, y)
        │
        ▼
ImageAnalysisContainer.vue
  onCanvasClick(x, y):
    getPixelOklchUseCase(imageData, x, y) を呼ぶ
        │
        ▼
application/useCase/getPixelOklchUseCase.ts
  ① getPixelAt(imageData, x, y)    [infrastructure/pixelReader.ts]
     境界チェック → i = (y * width + x) * 4
     → data[i], data[i+1], data[i+2] で RGB 取得
  ② sRGB(0-255) → sRGB(0-1) に正規化
     → Culori: formatHex() → oklch() で OKLCH に変換
  ③ createOklch(L, C, h)           [domain/oklch.ts]
     → OklchValue { L, C, h } ドメインオブジェクト生成
        │
        ▼
PixelOklchResult.vue
  L: 0.7234  C: 0.1542  h: 142.50
  (明度)      (彩度)      (色相)
```

## レイヤードアーキテクチャ

```
┌─────────────────────────────────────────────────┐
│  Presentation (features/)                        │
│  Vue コンポーネント・UI・ユーザー操作             │
├─────────────────────────────────────────────────┤
│  Application (application/useCase/)              │
│  ユースケース：ワークフローの組み立て             │
├─────────────────────────────────────────────────┤
│  Infrastructure (infrastructure/)                │
│  外部 API: File API, Canvas API, Culori          │
├─────────────────────────────────────────────────┤
│  Domain (domain/)                                │
│  OklchValue 値オブジェクト                        │
├─────────────────────────────────────────────────┤
│  Core (core/)                                    │
│  Result<T,E> 型安全なエラーハンドリング           │
└─────────────────────────────────────────────────┘
```

依存の向き: Presentation → Application → Domain ← Infrastructure

---

## エラーハンドリング

全レイヤーで `Result<T, E>` パターンを使い、try-catch なしで型安全にエラーを伝搬する。

```
infrastructure → Result<値, エラー型>
    ↓
application (UseCase) → isFailure() でチェック → 失敗ならそのまま返す
    ↓
presentation → isFailure() でチェック → errorMessage に表示
```

| エラー型 | 発生元 | レイヤー |
|---|---|---|
| ImageLoadError (FileReadError, CanvasError, InvalidImage) | imageLoader.ts | Infrastructure |
| PixelReadError (OutOfBounds) | pixelReader.ts | Infrastructure |
| GetPixelOklchError (PixelReadError, ConversionError) | getPixelOklchUseCase.ts | Application |

---

## 状態管理

グローバルストア（Pinia 等）は使わず、ImageAnalysisContainer 内の `ref()` だけでローカルに管理。

| 状態 | 型 | 役割 |
|---|---|---|
| `imageDataRef` | `ref<ImageData \| null>` | 読み込んだ画像のピクセルデータ |
| `errorMessage` | `ref<string \| null>` | エラーメッセージ |
| `pixelOklch` | `ref<OklchValue \| null>` | 選択ピクセルの OKLCH 値 |
| `loadProgress` | `ref<'idle' \| 'loading' \| 'done' \| 'error'>` | 画像読み込み状態 |
