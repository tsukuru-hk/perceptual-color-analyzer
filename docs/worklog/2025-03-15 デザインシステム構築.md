# 2025-03-15 デザインシステム構築

Tailwind CSS + shadcn-vue スタイルの UI コンポーネント基盤を構築し、既存の画像解析ページに統合した。

---

## 1. 技術選定

### 検討内容

- ユーザーの希望: Hero UI のようなモダン・スタイリッシュなデザイン + Lottie アニメーション
- Hero UI は React 専用のため Vue 3 では使用不可

### 採用した構成

| ライブラリ | バージョン | 役割 |
|-----------|-----------|------|
| Tailwind CSS v4 | `@tailwindcss/vite` | ユーティリティファーストのスタイリング |
| radix-vue | v1.x | アクセシブルなプリミティブ (Tabs, Tooltip, Slider) |
| class-variance-authority (CVA) | - | バリアント管理 (Button, Badge, Toast) |
| clsx + tailwind-merge | - | 条件付きクラス結合 (`cn()` ユーティリティ) |
| lucide-vue-next | - | アイコン |
| vue-router | - | ページルーティング |

### 選定理由

- **shadcn-vue スタイル**: Hero UI に最も近いミニマル・モダンな雰囲気。コピペ型でカスタマイズ自由度が最高
- **Tailwind CSS**: クラスベースで微調整しやすい。shadcn-vue の前提
- **radix-vue**: アクセシビリティを自前実装せずに済む。Vue 3 対応の唯一の headless UI ライブラリ

## 5. 既存ページへの統合

### ImageAnalysisContainer.vue のリファクタ

**Before:**
- 素の `<input type="file">`
- 横並び panels (元画像 + 彩度マップ)
- `PixelOklchResult`: モノスペーステキストで OKLCH 値表示
- scoped CSS でスタイリング
- エラーはインラインの赤テキスト

**After:**
- `<DropZone>` で D&D 対応のファイルアップロード
- `<Tabs>` で「元画像 / 彩度マップ」タブ切替
- 彩度マップ下に `<Legend>` で凡例表示
- `<ColorSwatch>` で選択ピクセルの色を視覚的にプレビュー
- 3つの `<ColorValueRow>` で L / C / h をバー+数値+コピーボタンで表示
- OKLCH → CSS: `oklch(${L} ${C} ${h})` でネイティブ CSS カラー関数を使用
- `useToast()` で「画像を読み込みました」「エラー」等の通知
- scoped CSS 全削除 → Tailwind ユーティリティクラスに移行
