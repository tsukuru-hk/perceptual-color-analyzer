## 選定理由

### Vue 3 (Composition API)

- **目的**: 画像解析結果の表示・インタラクション（ファイル選択、キャンバスクリック、OKLCH 表示）をコンポーネントでまとめるため。
- **理由**: Composition API でロジックの再利用（composables）や型との相性がよい。本プロジェクトは SPA 単体で、ルーティングや大規模状態管理は不要なため、Vue 単体で足りる。
- **他候補との比較**: React でも同様のことは可能。Vue はテンプレートと script の分離で「見た目とロジック」が分かりやすく、小〜中規模の UI に適している。

### TypeScript

- **目的**: ドメイン（OKLCH 値）、ユースケースの入出力、Result 型を型で明示し、実行時エラーを減らす。
- **理由**: Culori の戻り値や ImageData などの Web API を型で扱いたい。既存の設計（層分離・Result 型）と相性がよい。

### Vite

- **目的**: 開発サーバー（HMR）と本番ビルド。
- **理由**: Vue 公式で推奨されている。ESM ネイティブで起動が速く、Vue 3 + TS の設定が公式プラグインで簡単。Webpack より設定が少なく、小規模プロジェクトに適している。

### Culori

- **目的**: sRGB（ピクセル）から OKLCH への変換、および将来的な色空間まわりの計算。
- **理由**: OKLCH / Oklab を標準でサポートし、変換・補間・フォーマットが一通り揃っている。JavaScript/TypeScript で広く使われており、メンテナンスが活発。自前で色空間変換を実装するより安全で保守しやすい。

### Cloudflare Pages（デプロイ想定）

- **目的**: 静的サイトとしてのホスティング。
- **理由**: Vue + Vite のビルド成果物は静的ファイルのため、Pages の「ビルドコマンド + 出力ディレクトリ」でそのままデプロイできる。無料枠で個人・小規模利用に足りる。

### TresJS + Three.js（3D 可視化・準備済）

- **目的**: OKLCH 色空間の 3D ガマット表示、ピクセルデータの 3D プロット。
- **理由**: Vue 3 のコンポーネント指向で WebGL を宣言的に扱え、Composition API と親和性が高い。実装は未着手。

### D3.js（グラフ・準備済）

- **目的**: L / C / h の分布ヒストグラムやカスタムチャート。
- **理由**: 数値のマッピング自由度が高く、色彩統計を正確に視覚化できる。実装は未着手。

### GSAP（アニメーション・準備済）

- **目的**: OKLCH 数値のカウントアップ、解析シーケンスのタイムライン、Canvas/3D の補間。
- **理由**: 数値と描画を同期した滑らかなアニメーション制御に適している。実装は未着手。

### Motion for Vue（UI アニメーション・準備済）

- **目的**: パネル・ボタン・モーダルなどの UI インタラクションアニメーション。
- **理由**: Vue 向けに直感的な API を提供。実装は未着手。

## 補足

- **バックエンド**: 現状なし。画像処理・色変換はすべてブラウザ側（Canvas API + Culori）で完結。
- **状態管理**: 現状は Pinia 等は未使用。コンポーネント内の ref と composables で足りる想定。機能が増えたら検討。
- **テスト**: 現時点でテストフレームワークは未導入。ユースケース・インフラの単体テストを入れる場合は Vitest 等の導入を検討する。
